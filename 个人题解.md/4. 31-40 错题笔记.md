## 31. 题号：15 -- 三数之和（中等）
### 31.1 暴力解法 -- 不取

- **思路**：**三重循环 -- $O(n^3)$** 

### 31.2 哈希表 + 去重 + 双重循环 -- $O(n^2)$

- **思路** ：哈希表存前两个数之和，然后找另一个数是三个数之和为 0，即 b + c = -a -- O(n^2)，哈希表需考虑去重，而双指针不用考虑去重

### 31.3 最优解法及分析
代码如下：
```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        vector<vector<int>> res;
        // 先排序，方便去重和剪枝
        sort(nums.begin(), nums.end());
        // 排序后剪枝
        if (nums[0] > 0 || nums.back() < 0) return res;

        int n = nums.size();  // 每次循环都调用 nums.size() 虽开销微小，但可避免重复计算
        for (int i = 0; i < n - 2; ++i) {     // 隐式判断了 nums.size() < 3 的情况
            int a = nums[i];            // 避免对此下标访问
            // 循环内因为 i 的移动，也可以优化剪枝
            if (a > 0) break;
            // 第一次去重，防止重复的 nums[i]
            if (i > 0 && a == nums[i - 1]) continue;

            int l = i + 1, r = n - 1;
            while (l < r) {
                int sum = a + nums[l] + nums[r];
                if (sum == 0) {
                    res.push_back({a, nums[l], nums[r]});
                    // 第二次去重，左指针重复去重
                    while (l < r && nums[l] == nums[l + 1]) ++l;
                    // 第三次去重，右指针重复去重
                    while (l < r && nums[r] == nums[r - 1]) --r;
                    ++l;
                    --r;
                } else if (sum < 0) {
                    ++l;
                } else {
                    --r;
                }
            }
        }
        return res;
    }
};
```

- **知识点**：**排序去重+双指针**；

- **时间复杂度 -- $O(n^2)$，空间复杂度 -- $O(1)$**

- **时间优化点**：
1. 排序后马上判断 **最小值是否大于 0** 或 **最大值是否小于 0**，避免后续不必要的循环;
2. `for (int i = 0; i < n - 2; ++i)` **隐式判断了 `n < 3` 的情况**;
3. 对于要 **重复访问的值** `nums.size()` 和 `nums[i]` 不妨直接用 int 大小的空间 `n` 和 `a` 记录，**用空间换时间**

## 32. 题号：11 -- 盛最多水的容器（中等）
### 32.1 暴力解法
```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        int max = 0;
        int n = height.size();
        for (int i = 0; i < n - 1; ++i) {
            for (int j = i + 1; j < n; ++j) {
                int currArea = (j - i) * min(height[i], height[j]);
                if (currArea > max) {
                    max = currArea;
                }
            }
        }
        return max;
    }
};
```

- **思路**：**双重循环 + 贪心**;

- **时间复杂度 -- $O(n^2)$**

### 32.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        int L = 0, R = height.size() - 1;
        int max_Area = 0;
        while (L < R) {
            int currArea = (R - L) * min(height[L], height[R]);
            max_Area = max(max_Area, currArea);
            if (height[L] < height[R]) {
                ++L;
            } else {
                --R;
            }
        }
        return max_Area;
    }
};
```

- **知识点**：**贪心**，**双指针**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **思路**：**双指针 + 贪心**;

- **关键点**：**每次更新 height 值小的指针**；

## 33. 题号：42 -- 接雨水（困难）
### 33.1 错误解法

- **思路**：固定左指针到一个非零元素并进栈，移动右指针，直到右指针继续右移是值递减时右指针停止移动并进栈，根据二指针之间各个 height 与 min(height[L], height[R]) 的差值求和，然后更新左指针到右指针的位置（通过栈），右指针从左指针的下一位置开始遍历;

- **错误点**：
**逻辑错误**：
1. **“右指针继续右移是值递减时右指针停止移动”**：接雨水需要寻找 **两侧较高、中间较低** 的凹陷区域。正确的单调栈做法是：**维护一个递减栈**，当当前高度 **大于** 栈顶高度时，才说明遇到了可能的右边界，从而弹出栈顶计算积水。而你描述的方向相反（递减时停止），这会导致无法正确识别凹陷。
2. **“更新左指针到右指针的位置（通过栈）”**：会跳过许多可能的积水区域。
3. **“固定左指针到一个非零元素并进栈”**：忽略高度为0的柱子在某些情况下也是必要的（例如两侧高、中间低但中间有0），直接跳过可能丢失有效凹陷。

### 33.2 正确解法一 -- 对撞指针
代码如下：
```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int n = height.size();
        if (n < 3) return 0;       // 没有 3 个柱子不能形成水坑
        int sum = 0;

        int LeftMax = 0, RightMax = 0;  // 左、右侧所见最高柱子
        int L = 0, R = n - 1;
        while (L < R) {
            LeftMax = max(LeftMax, height[L]);
            RightMax = max(RightMax, height[R]);

            if (LeftMax < RightMax) {
                sum += LeftMax - height[L];
                ++L;
            } else {
                sum += RightMax - height[R];
                --R;
            }
        }
        return sum;
    }
};
```

- **知识点**：**对撞指针（双指针）**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **思路**：只要找到 **左边最高柱子** 和 **右边最高柱子**，中间比二者矮的“坑”就可以“积水”;

### 33.3 正确解法二 -- 单调栈
代码如下：
```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int n = height.size();
        if (n < 3) return 0;
        int sum = 0;

        stack<int> s;           // 递减存
        for (int i = 0; i < n; ++i) {
            // 当前高度 > 栈顶元素高度 --> 说明栈顶元素位置处为坑
            while (!s.empty() && height[i] > height[s.top()]) {
                int bottom = s.top();       // 当前坑的高度，用于计算填水的高度     
                s.pop();                    // 取栈顶元素后要出栈
                if (s.empty()) break;       // 下面要取栈顶的值，要判断上面的 bottom 出栈后栈内是否还有元素
                int left = s.top();         // 左墙 left, 右墙 i
                int h = min(height[left], height[i]) - height[bottom];
                int w = i - left - 1;       // 坑内所有元素的填水高度必定相同，因为是一层一层填水的
                sum += w * h;
            }
            s.push(i);
        }
        return sum;
    }
};
```

- **知识点**：**单调栈**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(n)$**

- **思路**：一行一行地填，从 `height = 1` 这一行逐渐向 `height = 2、3、...` 逐行填充，故而坑内各个元素的 `w` 必相等;

### 34.2 正确解法 1 -- 二分查找
代码如下：
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        int n = numbers.size();
        for (int i = 0; i < n - 1; ++i) {
            int num1 = numbers[i];
            int left = i + 1, right = n - 1;
            while (left <= right) {
                int mid = (left + right) / 2;
                int num2 = target - num1;
                if (numbers[mid] == num2) {
                    return {i + 1, mid + 1};
                } else if (numbers[mid] > num2) {
                    right = mid - 1;
                } else {
                    left = mid + 1;
                }
            }
        }
        return {};
    }
};

```

- **知识点**：；

- **时间复杂度 -- $O(nlogn)$，空间复杂度 -- $O(1)$**

- **思路**：**固定 `numbers[i]`，在 `[i+1, n-1]` 中二分查找 `target - numbers[i]`**;

### 34.3 正确解法 2 -- 对撞指针
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        // if (numbers[0] > target) return {};         // 前面两个也可以用这个剪枝优化 -- 这个剪枝在第一个数为负数时会出现错误
        // 正确的剪枝
        int n = numbers.size();
        if (numbers[0] + numbers[1] > target) return {};
        if (numbers[n - 1] + numbers[n - 2] < target) return {};
        int left = 0, right = n - 1;
        while (left < right) {
            int sum = numbers[left] + numbers[right];
            if (sum == target) {
                return {left + 1, right + 1};
            } else if (sum > target) {  // 类似二分，不多做解释
                --right;
            } else {
                ++left;
            }
        }
        return {};
    }
};
```

- **知识点**：**对撞指针**；

- **思路**：**类似二分查找，但是直接验证左右指针之和是否为 `target`**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

## 35. 题号：18 -- 四数之和（中等）
### 35.1 错误解法
```cpp
class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        int n = nums.size();
        sort(nums.begin(), nums.end());
        // 剪枝
        if (nums[0] + nums[1] + nums[2] + nums[3] > target) return {};
        if (nums[n - 1] + nums[n - 2] + nums[n - 3] + nums[n - 4] < target) return {};

        vector<vector<int>> res;
        for (int i = 0; i < n - 3; ++i) {
            int num1 = nums[i];
            for (int j = i + 1; j < n - 2; ++j) {
                int num2 = nums[j];
                int left = j + 1, right = n - 1;
                while (left < right) {
                    int sum = num1 + num2 + nums[left] + nums[right];
                    if (sum == target) {
                        res.push_back({num1, num2, nums[left], nums[right]});
                    } else if (sum > target) {
                        --right;
                    } else {
                        ++left;
                    }
                }
            }
        }
        return res;
    }
};
```

- **思路**：**固定 `nums[i]` + 三数之和 -- $O(n^3)$**;

- **错误点**：
1. **忘记去重了**；
2. **`sum = target`时，忘记更新 `left` 和 `right`**；  

### 35.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        int n = nums.size();
        if (n < 4) return {};
        sort(nums.begin(), nums.end());
        // 第一次剪枝：先排序，再剪枝
        if ((long long)nums[n - 1] + nums[n - 2] + nums[n - 3] + nums[n - 4] < target) return {};

        vector<vector<int>> res;
        for (int i = 0; i < n - 3; ++i) {
            int num1 = nums[i];
            // 第二次剪枝，注意四数之和防溢出
            // 剪枝 1：当前最小的四个数之和 > target，则任取其他三个数，四数之和均大于 target，直接退出循环
            if ((long long)num1 + nums[i + 1] + nums[i + 2] + nums[i + 3] > target) break;
            // 剪枝 2：当前 i 加上最大的三个数之和 < target，则当前 i 偏小，下面的循环肯定找不到三个数使四数之和等于 target，则退出本次循环
            if ((long long)num1 + nums[n - 1] + nums[n - 2] + nums[n - 3] < target) continue;
            // 第一次去重
            if (i > 0 && num1 == nums[i - 1]) continue;

            for (int j = i + 1; j < n - 2; ++j) {
                int num2 = nums[j];
                // 第三次剪枝
                // 剪枝 1
                if ((long long)num1 + num2 + nums[j + 1] + nums[j + 2] > target) break;
                // 剪枝 2
                if ((long long)num1 + num2 + nums[n - 1] + nums[n - 2] < target) continue;
                // 第二次去重
                if (j > i + 1 && num2 == nums[j - 1]) continue;

                int left = j + 1, right = n - 1;
                while (left < right) {
                    long long sum = (long long)num1 + num2 + nums[left] + nums[right];
                    if (sum == target) {
                        res.push_back({num1, num2, nums[left], nums[right]});
                        // 第三层去重 -- left
                        while (left < right && nums[left] == nums[left + 1]) ++left;
                        while (left < right && nums[right] == nums[right - 1]) --right;
                        // 更新 left 和 right
                        ++left;
                        --right;
                    } else if (sum > target) {  // 因为 sum != target，所以不用去重，只用更新
                        --right;
                    } else {
                        ++left;
                    }
                }
            }
        }
        return res;
    }
};
```

- **知识点**：**`ksum` 通过递归降至 `(k-1)sum`**，**四层去重**，**三次剪枝优化**；

- **时间复杂度 -- $O(n^3)$(排序 O(nlogn))，空间复杂度 -- $O(1)$**;

- **关键点**：
1. **计算四数之和是应该先把一个数转化为 `long long` 再求和，否则先求和就直接溢出 `INT_MAX` 了**；
2. **三次剪枝**：第一次是 **排序后最大四个数之和小于 `target`**，说明任取四个数之和均小于 `target`，数组中不存在 `四数之和 = target`，直接返回空向量；第二次是 **外层循环对于 `i` 的两次剪枝**，一是 **当前最小的四个数之和 > target**，则任取其他三个数，四数之和均大于 target，**直接退出循环（`break`）**，二是 **当前 i 加上最大的三个数之和 < target**，则当前 i 偏小，下面的循环肯定找不到三个数使四数之和等于 target，则 **退出本次循环（`continue`）**；第三次是 **内层循环对于 `j` 的两次剪枝**，同 `i`；

> **三次剪枝优化，进一步提升时间效率**

3. **四层去重**：第一层是 **外层循环对 `i` 去重**，如果当前 `nums[i]` 和前一个值相同，则退出本次循环（**不能用自增，否则会跳过数**）；第二层是 **内层循环对 `j` 去重**，同上；第三、四次是 **只有当 `sum == target` 时，才对左右指针去重，且之后要更新左右指针**，而 **当 `sum != target` 时，不用去重，只更新左右指针；

> **四次去重，避免重复的向量在数组中多次出现**

## 36. 题号：3 -- 无重复字符的最长子串（中等）
### 36.1 错误解法

- **思路**：貌似字符串匹配和这个挺像的，而字符串匹配有两种解法：1.暴力/朴素解法；2.KMP 算法(next[j]数组)，如何应用;

- **错误点**：**此题与 KMP 无关**，且代码未实现；

### 36.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        // 用 vector 维护哈希表检查重复，存取之前的值出现的下标
        vector<int> pos(128, -1);      // 标准 ASCII 码 128 个字符，扩展一些特殊符号、制表符或带音标的字母后为 256 个，-1 表示之前没有出现过
        int left = 0, res = 0;

        for (int right = 0; right < s.length(); ++right) {
            char c = s[right];
            // 如果当前 right 所指字符之前出现过，则寻找无重复字符的下一个子串
            if (pos[c] >= left) {
                left = pos[c] + 1;
            }
            // 更新 c 的最新位置(right 所指)
            pos[c] = right;
            // 更新最长长度
            res = max(res, right - left + 1);
        }
        return res;
    }
};
```

- **知识点**：**滑动窗口(同向双指针)**，**哈希表**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$（vector 哈希表）**

- **思路**：利用哈希表存之前出现过的值的 `index`，便于 **`left` 左指针指向下一个无重复字符的子串的第一个字符**；**右指针 `right` 不断向后探索**，**滑动窗口 `res` 长度永远在增加或不变，维护无重复字符的子串**;
