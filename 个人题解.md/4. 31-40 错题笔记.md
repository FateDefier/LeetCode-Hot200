## 31. 题号：15 -- 三数之和（中等）
### 31.1 暴力解法 -- 不取

- **思路**：**三重循环 -- $O(n^3)$** 

### 31.2 哈希表 + 去重 + 双重循环 -- $O(n^2)$

- **思路** ：哈希表存前两个数之和，然后找另一个数是三个数之和为 0，即 b + c = -a -- O(n^2)，哈希表需考虑去重，而双指针不用考虑去重

### 31.3 最优解法及分析
代码如下：
```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        vector<vector<int>> res;
        // 先排序，方便去重和剪枝
        sort(nums.begin(), nums.end());
        // 排序后剪枝
        if (nums[0] > 0 || nums.back() < 0) return res;

        int n = nums.size();  // 每次循环都调用 nums.size() 虽开销微小，但可避免重复计算
        for (int i = 0; i < n - 2; ++i) {     // 隐式判断了 nums.size() < 3 的情况
            int a = nums[i];            // 避免对此下标访问
            // 循环内因为 i 的移动，也可以优化剪枝
            if (a > 0) break;
            // 第一次去重，防止重复的 nums[i]
            if (i > 0 && a == nums[i - 1]) continue;

            int l = i + 1, r = n - 1;
            while (l < r) {
                int sum = a + nums[l] + nums[r];
                if (sum == 0) {
                    res.push_back({a, nums[l], nums[r]});
                    // 第二次去重，左指针重复去重
                    while (l < r && nums[l] == nums[l + 1]) ++l;
                    // 第三次去重，右指针重复去重
                    while (l < r && nums[r] == nums[r - 1]) --r;
                    ++l;
                    --r;
                } else if (sum < 0) {
                    ++l;
                } else {
                    --r;
                }
            }
        }
        return res;
    }
};
```

- **知识点**：**排序去重+双指针**；

- **时间复杂度 -- $O(n^2)$，空间复杂度 -- $O(1)$**

- **时间优化点**：
1. 排序后马上判断 **最小值是否大于 0** 或 **最大值是否小于 0**，避免后续不必要的循环;
2. `for (int i = 0; i < n - 2; ++i)` **隐式判断了 `n < 3` 的情况**;
3. 对于要 **重复访问的值** `nums.size()` 和 `nums[i]` 不妨直接用 int 大小的空间 `n` 和 `a` 记录，**用空间换时间**

## 32. 题号：11 -- 盛最多水的容器（中等）
### 32.1 暴力解法
```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        int max = 0;
        int n = height.size();
        for (int i = 0; i < n - 1; ++i) {
            for (int j = i + 1; j < n; ++j) {
                int currArea = (j - i) * min(height[i], height[j]);
                if (currArea > max) {
                    max = currArea;
                }
            }
        }
        return max;
    }
};
```

- **思路**：**双重循环 + 贪心**;

- **时间复杂度 -- $O(n^2)$**

### 32.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        int L = 0, R = height.size() - 1;
        int max_Area = 0;
        while (L < R) {
            int currArea = (R - L) * min(height[L], height[R]);
            max_Area = max(max_Area, currArea);
            if (height[L] < height[R]) {
                ++L;
            } else {
                --R;
            }
        }
        return max_Area;
    }
};
```

- **知识点**：**贪心**，**双指针**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **思路**：**双指针 + 贪心**;

- **关键点**：**每次更新 height 值小的指针**；

## 33. 题号：42 -- 接雨水（困难）
### 33.1 错误解法

- **思路**：固定左指针到一个非零元素并进栈，移动右指针，直到右指针继续右移是值递减时右指针停止移动并进栈，根据二指针之间各个 height 与 min(height[L], height[R]) 的差值求和，然后更新左指针到右指针的位置（通过栈），右指针从左指针的下一位置开始遍历;

- **错误点**：
**逻辑错误**：
1. **“右指针继续右移是值递减时右指针停止移动”**：接雨水需要寻找 **两侧较高、中间较低** 的凹陷区域。正确的单调栈做法是：**维护一个递减栈**，当当前高度 **大于** 栈顶高度时，才说明遇到了可能的右边界，从而弹出栈顶计算积水。而你描述的方向相反（递减时停止），这会导致无法正确识别凹陷。
2. **“更新左指针到右指针的位置（通过栈）”**：会跳过许多可能的积水区域。
3. **“固定左指针到一个非零元素并进栈”**：忽略高度为0的柱子在某些情况下也是必要的（例如两侧高、中间低但中间有0），直接跳过可能丢失有效凹陷。

### 33.2 正确解法一 -- 对撞指针
代码如下：
```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int n = height.size();
        if (n < 3) return 0;       // 没有 3 个柱子不能形成水坑
        int sum = 0;

        int LeftMax = 0, RightMax = 0;  // 左、右侧所见最高柱子
        int L = 0, R = n - 1;
        while (L < R) {
            LeftMax = max(LeftMax, height[L]);
            RightMax = max(RightMax, height[R]);

            if (LeftMax < RightMax) {
                sum += LeftMax - height[L];
                ++L;
            } else {
                sum += RightMax - height[R];
                --R;
            }
        }
        return sum;
    }
};
```

- **知识点**：**对撞指针（双指针）**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **思路**：只要找到 **左边最高柱子** 和 **右边最高柱子**，中间比二者矮的“坑”就可以“积水”;
