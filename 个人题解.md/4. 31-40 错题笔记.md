## 31. 题号：15 -- 三数之和（中等）
### 31.1 暴力解法 -- 不取

- **思路**：**三重循环 -- $O(n^3)$** 

### 31.2 哈希表 + 去重 + 双重循环 -- $O(n^2)$

- **思路** ：哈希表存前两个数之和，然后找另一个数是三个数之和为 0，即 b + c = -a -- O(n^2)，哈希表需考虑去重，而双指针不用考虑去重

### 31.3 最优解法及分析
代码如下：
```cpp
class Solution {
public:
    vector<vector<int>> threeSum(vector<int>& nums) {
        vector<vector<int>> res;
        // 先排序，方便去重和剪枝
        sort(nums.begin(), nums.end());
        // 排序后剪枝
        if (nums[0] > 0 || nums.back() < 0) return res;

        int n = nums.size();  // 每次循环都调用 nums.size() 虽开销微小，但可避免重复计算
        for (int i = 0; i < n - 2; ++i) {     // 隐式判断了 nums.size() < 3 的情况
            int a = nums[i];            // 避免对此下标访问
            // 循环内因为 i 的移动，也可以优化剪枝
            if (a > 0) break;
            // 第一次去重，防止重复的 nums[i]
            if (i > 0 && a == nums[i - 1]) continue;

            int l = i + 1, r = n - 1;
            while (l < r) {
                int sum = a + nums[l] + nums[r];
                if (sum == 0) {
                    res.push_back({a, nums[l], nums[r]});
                    // 第二次去重，左指针重复去重
                    while (l < r && nums[l] == nums[l + 1]) ++l;
                    // 第三次去重，右指针重复去重
                    while (l < r && nums[r] == nums[r - 1]) --r;
                    ++l;
                    --r;
                } else if (sum < 0) {
                    ++l;
                } else {
                    --r;
                }
            }
        }
        return res;
    }
};
```

- **知识点**：**排序去重+双指针**；

- **时间复杂度 -- $O(n^2)$，空间复杂度 -- $O(1)$**

- **时间优化点**：
1. 排序后马上判断 **最小值是否大于 0** 或 **最大值是否小于 0**，避免后续不必要的循环;
2. `for (int i = 0; i < n - 2; ++i)` **隐式判断了 `n < 3` 的情况**;
3. 对于要 **重复访问的值** `nums.size()` 和 `nums[i]` 不妨直接用 int 大小的空间 `n` 和 `a` 记录，**用空间换时间**

## 32. 题号：11 -- 盛最多水的容器（中等）
### 32.1 暴力解法
```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        int max = 0;
        int n = height.size();
        for (int i = 0; i < n - 1; ++i) {
            for (int j = i + 1; j < n; ++j) {
                int currArea = (j - i) * min(height[i], height[j]);
                if (currArea > max) {
                    max = currArea;
                }
            }
        }
        return max;
    }
};
```

- **思路**：**双重循环 + 贪心**;

- **时间复杂度 -- $O(n^2)$**

### 32.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    int maxArea(vector<int>& height) {
        int L = 0, R = height.size() - 1;
        int max_Area = 0;
        while (L < R) {
            int currArea = (R - L) * min(height[L], height[R]);
            max_Area = max(max_Area, currArea);
            if (height[L] < height[R]) {
                ++L;
            } else {
                --R;
            }
        }
        return max_Area;
    }
};
```

- **知识点**：**贪心**，**双指针**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **思路**：**双指针 + 贪心**;

- **关键点**：**每次更新 height 值小的指针**；

## 33. 题号：42 -- 接雨水（困难）
### 33.1 错误解法

- **思路**：固定左指针到一个非零元素并进栈，移动右指针，直到右指针继续右移是值递减时右指针停止移动并进栈，根据二指针之间各个 height 与 min(height[L], height[R]) 的差值求和，然后更新左指针到右指针的位置（通过栈），右指针从左指针的下一位置开始遍历;

- **错误点**：
**逻辑错误**：
1. **“右指针继续右移是值递减时右指针停止移动”**：接雨水需要寻找 **两侧较高、中间较低** 的凹陷区域。正确的单调栈做法是：**维护一个递减栈**，当当前高度 **大于** 栈顶高度时，才说明遇到了可能的右边界，从而弹出栈顶计算积水。而你描述的方向相反（递减时停止），这会导致无法正确识别凹陷。
2. **“更新左指针到右指针的位置（通过栈）”**：会跳过许多可能的积水区域。
3. **“固定左指针到一个非零元素并进栈”**：忽略高度为0的柱子在某些情况下也是必要的（例如两侧高、中间低但中间有0），直接跳过可能丢失有效凹陷。

### 33.2 正确解法一 -- 对撞指针
代码如下：
```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int n = height.size();
        if (n < 3) return 0;       // 没有 3 个柱子不能形成水坑
        int sum = 0;

        int LeftMax = 0, RightMax = 0;  // 左、右侧所见最高柱子
        int L = 0, R = n - 1;
        while (L < R) {
            LeftMax = max(LeftMax, height[L]);
            RightMax = max(RightMax, height[R]);

            if (LeftMax < RightMax) {
                sum += LeftMax - height[L];
                ++L;
            } else {
                sum += RightMax - height[R];
                --R;
            }
        }
        return sum;
    }
};
```

- **知识点**：**对撞指针（双指针）**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **思路**：只要找到 **左边最高柱子** 和 **右边最高柱子**，中间比二者矮的“坑”就可以“积水”;

### 33.3 正确解法二 -- 单调栈
代码如下：
```cpp
class Solution {
public:
    int trap(vector<int>& height) {
        int n = height.size();
        if (n < 3) return 0;
        int sum = 0;

        stack<int> s;           // 递减存
        for (int i = 0; i < n; ++i) {
            // 当前高度 > 栈顶元素高度 --> 说明栈顶元素位置处为坑
            while (!s.empty() && height[i] > height[s.top()]) {
                int bottom = s.top();       // 当前坑的高度，用于计算填水的高度     
                s.pop();                    // 取栈顶元素后要出栈
                if (s.empty()) break;       // 下面要取栈顶的值，要判断上面的 bottom 出栈后栈内是否还有元素
                int left = s.top();         // 左墙 left, 右墙 i
                int h = min(height[left], height[i]) - height[bottom];
                int w = i - left - 1;       // 坑内所有元素的填水高度必定相同，因为是一层一层填水的
                sum += w * h;
            }
            s.push(i);
        }
        return sum;
    }
};
```

- **知识点**：**单调栈**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(n)$**

- **思路**：一行一行地填，从 `height = 1` 这一行逐渐向 `height = 2、3、...` 逐行填充，故而坑内各个元素的 `w` 必相等;

## 34. 题号：167 -- 两数之和 II - 输入有序数组（中等）

```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        unordered_map<int, int> mp;         // value -- index
        int n = numbers.size();
        for (int i = 1; i < n; ++i) {
            int need = target - numbers[i];
            if (mp.find(need) != mp.end()) {
                return {mp[need], i};
            }
            mp[numbers[i]] = i; 
        }
        return {};
    }
};
```

- **错误点**：**哈希表空间复杂度为 $O(n)$，不满足题目要求的 $O(1)$**;

### 34.2 正确解法 1 -- 二分查找
代码如下：
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        int n = numbers.size();
        for (int i = 0; i < n - 1; ++i) {
            int num1 = numbers[i];
            int left = i + 1, right = n - 1;
            while (left <= right) {
                int mid = (left + right) / 2;
                int num2 = target - num1;
                if (numbers[mid] == num2) {
                    return {i + 1, mid + 1};
                } else if (numbers[mid] > num2) {
                    right = mid - 1;
                } else {
                    left = mid + 1;
                }
            }
        }
        return {};
    }
};

```

- **知识点**：；

- **时间复杂度 -- $O(nlogn)$，空间复杂度 -- $O(1)$**

- **思路**：**固定 `numbers[i]`，在 `[i+1, n-1]` 中二分查找 `target - numbers[i]`**;

### 34.3 正确解法 2 -- 对撞指针
```cpp
class Solution {
public:
    vector<int> twoSum(vector<int>& numbers, int target) {
        // if (numbers[0] > target) return {};         // 前面两个也可以用这个剪枝优化 -- 这个剪枝在第一个数为负数时会出现错误
        // 正确的剪枝
        int n = numbers.size();
        if (numbers[0] + numbers[1] > target) return {};
        if (numbers[n - 1] + numbers[n - 2] < target) return {};
        int left = 0, right = n - 1;
        while (left < right) {
            int sum = numbers[left] + numbers[right];
            if (sum == target) {
                return {left + 1, right + 1};
            } else if (sum > target) {  // 类似二分，不多做解释
                --right;
            } else {
                ++left;
            }
        }
        return {};
    }
};
```

- **知识点**：**对撞指针**；

- **思路**：**类似二分查找，但是直接验证左右指针之和是否为 `target`**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

## 35. 题号：18 -- 四数之和（中等）
### 35.1 错误解法
```cpp
class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        int n = nums.size();
        sort(nums.begin(), nums.end());
        // 剪枝
        if (nums[0] + nums[1] + nums[2] + nums[3] > target) return {};
        if (nums[n - 1] + nums[n - 2] + nums[n - 3] + nums[n - 4] < target) return {};

                int num2 = nums[j];
                int left = j + 1, right = n - 1;
                while (left < right) {
                    int sum = num1 + num2 + nums[left] + nums[right];
                    if (sum == target) {
                        res.push_back({num1, num2, nums[left], nums[right]});
                    } else if (sum > target) {
                        --right;
                    } else {
                        ++left;
                    }
                }
            }
        }
        return res;
    }

```

- **思路**：**固定 `nums[i]` + 三数之和 -- $O(n^3)$**;

- **错误点**：
1. **忘记去重了**；
2. **`sum = target`时，忘记更新 `left` 和 `right`**；  

### 35.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    vector<vector<int>> fourSum(vector<int>& nums, int target) {
        int n = nums.size();
        if (n < 4) return {};
        sort(nums.begin(), nums.end());
        // 第一次剪枝：先排序，再剪枝
        if ((long long)nums[n - 1] + nums[n - 2] + nums[n - 3] + nums[n - 4] < target) return {};

        vector<vector<int>> res;
        for (int i = 0; i < n - 3; ++i) {
            int num1 = nums[i];
            // 第二次剪枝，注意四数之和防溢出
            // 剪枝 1：当前最小的四个数之和 > target，则任取其他三个数，四数之和均大于 target，直接退出循环
            if ((long long)num1 + nums[i + 1] + nums[i + 2] + nums[i + 3] > target) break;
            // 剪枝 2：当前 i 加上最大的三个数之和 < target，则当前 i 偏小，下面的循环肯定找不到三个数使四数之和等于 target，则退出本次循环
            if ((long long)num1 + nums[n - 1] + nums[n - 2] + nums[n - 3] < target) continue;
            // 第一次去重
            if (i > 0 && num1 == nums[i - 1]) continue;

            for (int j = i + 1; j < n - 2; ++j) {
                int num2 = nums[j];
                // 第三次剪枝
                // 剪枝 1
                if ((long long)num1 + num2 + nums[j + 1] + nums[j + 2] > target) break;
                // 剪枝 2
                if ((long long)num1 + num2 + nums[n - 1] + nums[n - 2] < target) continue;
                // 第二次去重
                if (j > i + 1 && num2 == nums[j - 1]) continue;

                int left = j + 1, right = n - 1;
                while (left < right) {
                    long long sum = (long long)num1 + num2 + nums[left] + nums[right];
                    if (sum == target) {
                        res.push_back({num1, num2, nums[left], nums[right]});
                        // 第三层去重 -- left
                        while (left < right && nums[left] == nums[left + 1]) ++left;
                        while (left < right && nums[right] == nums[right - 1]) --right;
                        // 更新 left 和 right
                        ++left;
                        --right;
                    } else if (sum > target) {  // 因为 sum != target，所以不用去重，只用更新
                        --right;
                    } else {
                        ++left;
                    }
                }
            }
        }
        return res;
    }
};
```

- **知识点**：**`ksum` 通过递归降至 `(k-1)sum`**，**四层去重**，**三次剪枝优化**；

- **时间复杂度 -- $O(n^3)$(排序 O(nlogn))，空间复杂度 -- $O(1)$**;

- **关键点**：
1. **计算四数之和是应该先把一个数转化为 `long long` 再求和，否则先求和就直接溢出 `INT_MAX` 了**；
2. **三次剪枝**：第一次是 **排序后最大四个数之和小于 `target`**，说明任取四个数之和均小于 `target`，数组中不存在 `四数之和 = target`，直接返回空向量；第二次是 **外层循环对于 `i` 的两次剪枝**，一是 **当前最小的四个数之和 > target**，则任取其他三个数，四数之和均大于 target，**直接退出循环（`break`）**，二是 **当前 i 加上最大的三个数之和 < target**，则当前 i 偏小，下面的循环肯定找不到三个数使四数之和等于 target，则 **退出本次循环（`continue`）**；第三次是 **内层循环对于 `j` 的两次剪枝**，同 `i`；

> **三次剪枝优化，进一步提升时间效率**

3. **四层去重**：第一层是 **外层循环对 `i` 去重**，如果当前 `nums[i]` 和前一个值相同，则退出本次循环（**不能用自增，否则会跳过数**）；第二层是 **内层循环对 `j` 去重**，同上；第三、四次是 **只有当 `sum == target` 时，才对左右指针去重，且之后要更新左右指针**，而 **当 `sum != target` 时，不用去重，只更新左右指针；

> **四次去重，避免重复的向量在数组中多次出现**

## 36. 题号：3 -- 无重复字符的最长子串（中等）
### 36.1 错误解法

- **思路**：貌似字符串匹配和这个挺像的，而字符串匹配有两种解法：1.暴力/朴素解法；2.KMP 算法(next[j]数组)，如何应用;

- **错误点**：**此题与 KMP 无关**，且代码未实现；

### 36.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    int lengthOfLongestSubstring(string s) {
        // 用 vector 维护哈希表检查重复，存取之前的值出现的下标
        vector<int> pos(128, -1);      // 标准 ASCII 码 128 个字符，扩展一些特殊符号、制表符或带音标的字母后为 256 个，-1 表示之前没有出现过
        int left = 0, res = 0;

        for (int right = 0; right < s.length(); ++right) {
            char c = s[right];
            // 如果当前 right 所指字符之前出现过，则寻找无重复字符的下一个子串
            if (pos[c] >= left) {
                left = pos[c] + 1;
            }
            // 更新 c 的最新位置(right 所指)
            pos[c] = right;
            // 更新最长长度
            res = max(res, right - left + 1);
        }
        return res;
    }
};
```

- **知识点**：**滑动窗口(同向双指针)**，**哈希表**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$（vector 哈希表）**

- **思路**：利用哈希表存之前出现过的值的 `index`，便于 **`left` 左指针指向下一个无重复字符的子串的第一个字符**；**右指针 `right` 不断向后探索**，**滑动窗口 `res` 长度永远在增加或不变，维护无重复字符的子串**;

## 37. 题号：438 -- 找到字符串中所有字母异位词（中等）
### 37.1 错误解法
```cpp
class Solution {
public:
    vector<int> findAnagrams(string s, string p) {
        if (s.length() < p.length()) return {};

        vector<int> res;
        vector<int> pos(26, -1);        // 哈希表，-1 表示在 p 中不存在，1 表示存在
        for (int i = 0; i < p.length(); ++i) {
            char c = p[i];
            pos[(int)c - (int)'a'] = 1;
        }
        for (int L = 0; L <= s.length() - p.length(); ++L) {
            int R = L + p.length() - 1;
            int i = L;              // 遍历窗口，看滑动窗口元素是否在 p 中
            while (i <= R) {
                if (pos[(int)s[i] - (int)'a'] == -1) {
                    break;
                } else {
                    ++i;
                    continue;
                }
            }
            if (i == R) {
                res.push_back(L);
            }
        }
        return res;
    }
};
```

- **思路**：题目明确 `s` 和 `p` 只含小写字母，且异位词很容易想到哈希表存 `p` 中出现过的元素用于跟 `s` 进行比较，找 `p` 的异位词，只需要固定滑动窗口的大小等于 `p.length()`，判断该窗口是否出现了 `p` 中的所有字符;

- **错误点**：
1. **最严重的逻辑错误**：我的代码表达的意思：**只要滑动窗口的字符在 `p` 中出现过，就是异位词，应该是字符在 `p` 中出现多少次，在滑动窗口中就出现多少次**；(**这里可用哈希表存字符出现次数**)

> 反例：`s="aaa", p="ab"`。你的代码会认为 `"aa"` 是异位词（因为 `'a'` 在 p 中存在），但实际上 `"aa"` 缺少 `'b'` 且 'a' 数量不对

2. **边界判断错误**：`while` 循环结束后，**`i` 会自增到大于 `R`**；

3. **时间复杂度退化**：滑动窗口的精髓就在于 **增量更新**，正常时间复杂度为 $O(s)$，而代码退化到 $O(s \cdot p)$

### 37.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    vector<int> findAnagrams(string s, string p) {
        vector<int> res;
        int m = s.length(), n = p.length();
        if (m < n) return {};

        // need：p 中各字符出现的频率，即需要窗口出现的次数
        // window: 窗口各字符出现的频率
        vector<int> need(26, 0), window(26, 0);
        // 初始化 need
        for (auto c : p) ++need[c - 'a'];
        // 初始化 window(长度 = p.length())
        for (int i = 0; i < n; ++i) ++window[s[i] - 'a'];
        if (need == window) res.push_back(0);

        // 滑动窗口：左右边界动态移动，移出左边界，移入右边界，保证窗口大小不变
        for (int i = 0; i < m - n; ++i) {
            --window[s[i] - 'a'];           // 移出左边界
            ++window[s[i + n] - 'a'];       // 移入右边界
            if (need == window) res.push_back(i + 1);   // 注意这里是 i + 1，因为是先移动窗口，再判断
        }
        return res;
    }
};
```

- **知识点**：**滑动窗口**，**哈希表**；

- **时间复杂度 -- $O(26s)$（`need == window` 复杂度为 $O(26)$，可优化），空间复杂度 -- $O(1)$**

- **思路**：两个哈希表分别存 p 中字符出现的频率和滑动窗口出现的频率，二者相等即为 Anagram(异位词);

### 37.3 优化解法
```cpp
class Solution {
public:
    vector<int> findAnagrams(string s, string p) {
        int m = s.length(), n = p.length();
        if (m < n) return {};

        vector<int> diff(26, 0);    // 记录窗口与 p 之间有多少种字母频率不匹配
        int nonZero = 0;            // 记录 diff 中的非零元素，即不匹配的字符个数

        // 初始化 diff，先加 s[n](s 前 n 个元素)，再减 p[n]
        for (int i = 0; i < n; ++i) {
            ++diff[s[i] - 'a'];
            --diff[p[i] - 'a'];
        }
        // 初始化 nonZero
        for (int i = 0; i < 26; ++i) {
            if (diff[i] != 0) ++nonZero;
        }

        vector<int> res;
        if (nonZero == 0) res.push_back(0);

        // 滑动窗口
        for (int i = 0; i < m - n; ++i) {
            // 移出左边界
            int out = s[i] - 'a';
            // 如果当前左边字符为匹配，移出后就变为不匹配了
            if (diff[out] == 0) ++nonZero;
            // 如果当前左边字符为不匹配（前面的if判断了匹配的，到这就是不匹配），移出左边界后可能变成匹配
            --diff[out];
            if (diff[out] == 0) --nonZero;
            
            // 移入右边界
            int in = s[i + n] - 'a';
            if (diff[in] == 0) ++nonZero;
            ++diff[in];
            if (diff[in] == 0) --nonZero;

            if (nonZero == 0) res.push_back(i + 1);
        }
        return res;
    }
};
```

- **关键点**：
1. **`diff` 哈希表 记录窗口与 p 之间有多少种字母频率不匹配**（匹配相应值为 0，不匹配不为 0），**`nonZero` 记录不匹配字母的个数**，`nonZero = 0` 就找到了 Anagram(异位词)；
2. **移动左、有边界** 之前要先 **在数值被修改前，抢先记录下“即将丢失的匹配状态”**，从而保证 nonZero 计数器在任何时刻都精确等于 diff 数组中非零元素的真实个数；

## 38. 题号：76 -- 最小覆盖子串（中等）
### 38.1 错误解法

- **思路**：同上，但需调整，代码写不出来;

### 38.2 正确解法及分析
代码如下：
```cpp
class Solution {
private:
    // 把字母映射到 0 ~ 51 下标
    int index(char c) {
        return (c >= 'a') ? (c - 'a') : (c - 'A' + 26);
    }
public:
    string minWindow(string s, string t) {
        if (s.empty() || t.empty()) return "";
        int m = s.length(), n = t.length();
        if (m < n) return "";

        int diff[52] = {0};             // 0~25大写字母，26~51小写字母 = window - need
        int satisfied = 0;              // 当前窗口已满足的字符个数
        int totalNeed = 0;              // t 需要的字符个数

        // 初始化 diff 和 totalNeed -- 只记录 need
        for (char c : t) {
            if (diff[index(c)] == 0) ++totalNeed;       // 只记录字符种类数，故而判断 diff[index(c)] 是否为 0
            --diff[index(c)];
        }

        int minLen = INT_MAX, minLeft = 0;           // 初始化最短子串窗口长度及最短窗口子串的起始位置(左指针)
        int left = 0;                   // 左指针

        for (int right = 0; right < m; ++right) {
            // 右边界字符不断进入窗口，直至 satisfied == totalNeed
            if (diff[index(s[right])] == -1) ++satisfied;
            ++diff[index(s[right])];

            // 窗口包含 t 后，尝试缩小左边界以满足最短窗口
            while (satisfied == totalNeed) {
                // 更新长度
                if (right - left + 1 < minLen) {
                    minLen = right - left + 1;
                    minLeft = left;         // 关键点
                }
                // 缩小边界并更新 satisfied
                --diff[index(s[left])];
                if (diff[index(s[left])] == -1) --satisfied;
                ++left;
            }
        }
        return minLen == INT_MAX ? "" : s.substr(minLeft, minLen);
    }
};
```

- **知识点**：**滑动窗口**，**哈希表**，**差分数组+非零计数**；

- **时间复杂度 -- $O(m + n)$（左右指针最多 2m，实际不可能达到），空间复杂度 -- $O(1)$**

- **思路**：最小 window 要包含个数相同的 t 中的所有种类字符，且**可以多出其他字符**，要满足三个要求：
1. **种类相同：用 `satified == totalNeed` 解决**；
2. **个数相同：用差分数组 `diff == 0` 解决**；
3. **最小窗口：左指针 `left` 收缩左边界解决**;

- **关键点**：
1. **只记录左右指针，不在循环中进行子串拷贝，避免在双重循环中 $O(n^2)$**；
2. **只在 `diff[c] == -1` 时增加 `satisfied`**，因为 `satisfied` 和 `totalNeed` 都是记录字符种类数，在后面的自增中总会有自增到 `diff[index(s[right])] == -1` 的时候；
3. **必须要使用 `minLeft` 记录最短窗口的起始位置**，因为 `left` 在不断缩短左边界，如果前面找到了最短窗口那 left 反而没有记录；

## 39. 题号：39 -- 滑动窗口最大值（困难）
### 39.1 错误解法 -- 超时
```cpp
class Solution {
private:
    int maxValue(vector<int>& nums, int L, int R) {
        int max = nums[L];
        for (int i = L + 1; i <= R; ++i) {
            if (nums[i] > max) {
                max = nums[i];
            }
        }
        return max;
    }
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {

        if (n < k) return {};

        vector<int> res;

            int right = left + k - 1;
            res.push_back(maxValue(nums, left, right));
        }
        return res;
    }
};
```

- **思路**：手动控制左指针，`right = left + k - 1`，定义一个函数找两个指针之间数组的最大值;

- **时间复杂度 -- $O(n \cdot k)$，空间复杂度 -- $O(1)$**

### 39.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        int n = nums.size();
        if (n < k) return {};

        deque<int> dq;              // 存窗口内元素的递减序列(队头最大)，存 index
        vector<int> res;

        for (int i = 0; i < n; ++i) {
            // 1. 清理队头的过期元素：队头出队(有多个过期清理多个)
            while (!dq.empty() && dq.front() <= i - k) {
                dq.pop_front();
            }
            // 2. 清除队尾 <= nums[i] 的元素，它们 <= nums[i]，不可能为窗口的最大值(因为 nums[i] 已经 >= 它们)，这里比较时是值比较
            while (!dq.empty() && nums[dq.back()] <= nums[i]) {
                dq.pop_back();
            }
            // 3. nums[i] 入队
            dq.push_back(i);
            // 4. 判断是否达到窗口大小，达到则加入 res
            if (i + 1 >= k) res.push_back(nums[dq.front()]);    // 注意，dq 存的是下标，加入 res 的是 value
        }
        return res;
    }
};
```

- **知识点**：**单调双端队列**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(k)$**

- **STL `deque` 的内存访问模式对 CPU 缓存极不友好，手动实现 `deque` 可以优化时间**

- **思路**：看代码注释，思路很清晰;

### 39.3 优化解法
```cpp
class Solution {
public:
    vector<int> maxSlidingWindow(vector<int>& nums, int k) {
        int n = nums.size();
        if (n == 0 || k == 0) return {};
        if (n < k) return {};

        int dq[100000];          // 1 <= nums.length <= 10^5，存下标，存值在有重复值时无法判断删除哪一个保留在窗口的元素
        // int* dq = new int[n];       // 极致的空间优化：动态分配存储空间，但是 new 比直接定义数组时间效率更低，但是可以避免栈溢出的问题
        int head = 0, tail = 0;     // dq 的首尾指针，head 指向队头元素，tail 指向队尾元素的下一位置

        vector<int> res;
        res.reserve(n - k + 1);     // 极致的空间优化：预留空间，避免 push_back 扩容

        for (int i = 0; i < n; ++i) {
            // 1. 删除队头过期元素(逻辑删除)
            while (tail > head && dq[head] <= i - k) {
                ++head;
            }
            // 2. 剔除队尾 <= nums[i] 的元素(逻辑删除)
            while (tail > head && nums[dq[tail - 1]] <= nums[i]) {
                --tail;
            }
            // 3. 加入 i -- 存下标
            dq[tail++] = i;
            // 4. 判断窗口是否形成
            if (i + 1 >= k) {
                res.push_back(nums[dq[head]]);
            }
        }
        return res;
    }
};
```

- **优化点**：
1. 题目要求 1 <= nums.length <= $10^5$，定义 `int dq[100000]`，**直接定义数组比动态分配存储空间更快**；
2. **队列必须存下标不能存值**，因为存值在有重复值时无法判断删除哪一个元素，保留哪一个元素在窗口；

## 40. 题号：560 -- 和为 K 的子数组（中等）
### 40.1 错误解法
```cpp
class Solution {
public:
    int subarraySum(vector<int>& nums, int k) {
        int n = nums.size();
        long long preSum[n] = {0};
        unordered_map<long long, int> mp;   // value -- index

        preSum[0] = nums[0];
        for (int i = 1; i < n; ++i) {
            preSum[i] = preSum[i - 1] + nums[i];
        }

        mp[preSum[0]] = 0;
        int count = 0;
        for (int i = 1; i < n; ++i) {
            long long target = preSum[i] - k;
            if (mp.find(target) != mp.end()) {
                ++count;
            }
            mp[target] = i;
        }
        return count;
    }
};
```

- **思路**：前缀和，先求出数组截止到每个元素的前缀和存入数组中，然后看前缀和哪两个之间差值为 $k$，单纯这样做类似暴力解法，时间复杂度为 $O(n^2)$，故而借用哈希表可将时间复杂度优化到 $O(n)$，时间复杂度 $O(n)$;

- **错误点**：
1. **VLA 非标准且初始化错误**：`long long preSum[n] = {0};` 是 C 风格**变长数组（VLA，Variable Length Array）**，C++ 标准不支持，且当 `n` 为变量时不能直接用 `{0}` 初始化。应改用 `vector<long long> preSum(n, 0);`；
2. **哈希表语义错误（逻辑错误）**：本题需要记录的是 **前缀和出现的次数**，而非索引。因为同一个前缀和可能出现多次，每次都能构成一个合法子数组。
3. **遗漏边界情况**：未预先存入 mp[0] = 1，导致从下标 0 开始的合法子数组被漏统计。

### 40.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    int subarraySum(vector<int>& nums, int k) {
        unordered_map<long long, int> mp;   // preSum -- count
        mp.reserve(nums.size() + 1);        // 插入前预分配，避免 rehash
        mp[0] = 1;                          // 第 0 各元素前缀和为 0，未遍历数组时出现一次

        int count = 0;
        long long preSum = 0;       // 不用数组 O(n)，只用 O(1)
        for (int num : nums) {
            preSum += num;
            if (mp.find(preSum - k) != mp.end()) {
                count += mp[preSum - k];    // 哈希表存的是前缀和出现的次数，对应不同子数组的个数，这里 count 应该加它，而不是自增
            }
            ++mp[preSum];
        }
        return count;
    }
};
```

- **知识点**：**哈希表**，**前缀和**(`preSum = nums[0...i-1]`)；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **关键点**：
1. **哈希表 -- `pair<preSum, count>`**；
2. `mp.reserve(nums.size() + 1);`：插入前预分配，避免 rehash；
3. `count += mp[preSum - k];` 哈希表存的是前缀和出现的次数，对应不同子数组的个数，这里 **`count` 应该加哈希表中前缀和出现的次数（子数组数量）**，而不是自增；
