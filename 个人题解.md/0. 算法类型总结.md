# 一、C++ 语法相关
## 1. 各数据类型所占字节及表示范围

以下是基于 **Windows + MinGW/G++ (64位) + VSCode** 及主流 OJ（LeetCode/Codeforces/洛谷）的通用标准。现代 OJ 和 Windows G++ 均采用 **LP64/LLP64** 模型，数据宽度高度一致。

### 📊 核心数据类型速查表

| 类型 | 字节数 | 精确表示范围 | 数量级记忆 | 适用场景 |
| :--- | :---: | :--- | :--- | :--- |
| `bool` | 1 | `true` / `false` | - | 状态标记 |
| `char` | 1 | $[-128, 127]$ | $10^2$ | 字符、小值域计数 |
| `int` | 4 | $[-2^{31}, 2^{31}-1]$ | $\pm 2.1 \times 10^9$ | 索引、$N \leq 2\times10^5$ 的累加 |
| `unsigned int` | 4 | $[0, 2^{32}-1]$ | $4.2 \times 10^9$ | 哈希值、位运算、模 $2^{32}$ |
| `long long` | 8 | $[-2^{63}, 2^{63}-1]$ | $\pm 9.2 \times 10^{18}$ | 前缀和、大数累乘、$N \leq 10^6$ 的平方级运算 |
| `float` | 4 | $\approx \pm 3.4 \times 10^{38}$ | 有效精度仅 ~7 位 | **算法题极少用**，精度陷阱多 |
| `double` | 8 | $\approx \pm 1.7 \times 10^{308}$ | 有效精度 ~15 位 | 浮点计算、概率DP、几何题 |
| `size_t` | 8 | $[0, 2^{64}-1]$ | $1.8 \times 10^{19}$ | 容器大小、内存地址（无符号） |

> ⚠️ **关键注意**：Windows G++ 中 `long` 是 **4 字节**（与 `int` 相同），这与 Linux64 下 `long` 为 8 字节不同。**跨平台/OJ 代码中永远用 `long long` 代替 `long`**，避免移植时静默溢出。

## 2. C++ 的输入输出精度表示

C++ 设置精度其实比 C 的 `printf` 格式化更直观，核心只需记住两个操纵符。

### 🎯 常用精度控制速查

> ⚠️ **必带头文件**：`#include <iomanip>`(**IO Manipulators（输入输出操纵符）**)，否则 `setprecision` 编译报错。

| 需求 | C++ 写法 | 等价 C 写法 | 说明 |
| :--- | :--- | :--- | :--- |
| 保留 n 位小数 | `cout << fixed << setprecision(n) << x;` | `printf("%.nf", x);` | **最常用**，四舍五入 |
| 保留 n 位有效数字 | `cout << setprecision(n) << x;` | `printf("%.ng", x);` | 不加 `fixed` 时默认为有效数字 |
| 科学计数法 n 位 | `cout << scientific << setprecision(n) << x;` | `printf("%.ne", x);` | 大数/小数输出 |
| 恢复默认 | `cout.unsetf(ios::floatfield);` | - | 取消 fixed/scientific |

### 💡 关键注意事项

-   **`fixed` 是状态标志**：一旦设置，后续所有浮点输出都按固定小数位，直到手动取消。若只想要某一行精确，用完记得 `unsetf`。
```cpp
// 仅本行固定小数，之后恢复默认
cout << fixed << setprecision(n) << x << defaultfloat;
```
`defaultfloat（C++11）`可一键恢复默认浮点格式，比 `unsetf(ios::floatfield)` 更简洁安全
-   **整数不受影响**：`setprecision` 仅对浮点数生效，输出整数时无需担心被截断。
-   **与 IO 优化兼容**：精度设置与 `sync_with_stdio(false)` 完全正交，可同时使用。

## 3. `emplace_back + move` 和 `push_back`
这两个概念是现代 C++ 性能优化的基石，核心目的都是**消除不必要的深拷贝**。

### 1. `std::move`：所有权的“转移”而非“移动”
它本质上只是一个**类型转换**（将左值转为右值引用），告诉编译器：“这个对象我不要了，你可以直接偷走它的内部资源”。

-   **没有 move**：`vector.push_back(str)` → 触发 `string` 的**拷贝构造**，分配新内存并逐字节复制内容。
-   **有 move**：`vector.push_back(std::move(str))` → 触发 `string` 的**移动构造**，仅复制指针和长度，原 `str` 变为有效但未指定状态（通常为空）。
-   **⚠️ 关键认知**：move 本身不移动任何数据，它只是“允许”移动。真正的资源转移发生在移动构造函数中。

### 2. `emplace_back`：原地构造，省去临时对象
它与 `push_back` 的区别在于参数传递方式：

-   `push_back(obj)`：先构造/拷贝一个临时对象，再将其移入容器（即使有 move，也经历了一次临时对象的创建）。
-   `emplace_back(args...)`：将参数**完美转发**到容器内部，直接在目标内存位置上调用构造函数，**零临时对象**。

#### 💡 何时 emplace_back 真正优于 push_back？
| 场景 | push_back | emplace_back | 优势方 |
| :--- | :--- | :--- | :--- |
| 传入已有对象 | 1次 move/copy | 1次 move/copy | **平手** |
| 传入构造参数 | 1次临时构造 + 1次 move | **直接原地构造** | **emplace** |
| 复杂对象初始化 | 开销大 | 开销最小 | **emplace** |

```cpp
vector<string> v;
// 情况A：已有对象 → 两者等价
string s = "hello";
v.push_back(std::move(s));      // OK
v.emplace_back(std::move(s));   // OK，但无额外收益

// 情况B：现场构造 → emplace 胜出
v.push_back(string("world"));   // 构造临时string → move进vector
v.emplace_back("world");        // 直接在vector内存上构造string，零临时对象
```

### ⚠️ 常见误区提醒
1.  **不要对常量或基本类型用 move**：`int`, `char` 等 POD 类型 move 等同于 copy，反而可能阻碍编译器优化。
2.  **move 后勿再用原对象**：除非重新赋值，否则行为未定义（虽然标准保证“有效但未指定”，但业务逻辑上应视为已销毁）。
3.  **emplace_back 不支持隐式转换链过长**：如 `vector<vector<int>> v; v.emplace_back({1,2});` 在某些旧编译器可能失败，需显式类型。

# 二、算法类型总结

### 1.贪心算法 (Greedy)

- **特点**：每一步都做出当前状态下**局部最优**的选择，并希望这些局部最优能累积成全局最优。
- **决策依据**：只依赖当前信息，不回溯，不保存子问题结果。
- **适用条件**：问题具有**贪心选择性质**（全局最优可以通过局部最优得到）和**最优子结构**。
- **例子（方法一）**：  
  遍历到第 `i` 天时，我们只需要知道 `[0, i-1]` 天中的**最低价**。  
  为什么可以这么做？因为无论未来价格如何，**只要在今天卖出，买入价一定是过去最低点**。  
  这种“取过去最小”就是每一步的贪心选择。

### 2.动态规划 (Dynamic Programming)

- **特点**：将原问题拆解成**重叠子问题**，通过状态转移方程递推求解，通常需要保存中间状态。
- **决策依据**：综合考虑当前选择与子问题的最优解，可能从多个候选状态中取最优。
- **适用条件**：问题具有**最优子结构**和**重叠子问题**。
- **例子（方法二）**：  
  定义 `dp[i]` 为“以第 `i` 天结束时的最大累计利润（且允许空仓）”，则  
  `dp[i] = max(0, dp[i-1] + prices[i] - prices[i-1])`。  
  这里 `dp[i]` 依赖 `dp[i-1]` 和当天差价，是典型的一维 DP（Kadane 算法）。  
  DP 需要保存 `dp[i-1]`（一个状态变量），虽然没有显式数组，但状态转移的思想是 DP 的核心。

## 核心区别总结表

| 特性 | 贪心算法 | 动态规划 |
|------|----------|----------|
| **子问题关系** | 子问题独立或递推中只依赖常数个前驱 | 子问题可能重叠，常需保存多个历史状态 |
| **状态保存** | 通常只需少量变量（如 `min_price`） | 可能需要数组或矩阵（但一维 DP 也可压缩） |
| **决策过程** | 每步选当前最优，不做回退 | 比较多种可能性，选取全局最优 |
| **正确性证明** | 需要证明贪心选择性质 | 依靠状态转移方程和最优子结构 |
| **典型问题** | 股票一次交易、区间调度、霍夫曼编码 | 背包问题、最长公共子序列、编辑距离 |

### **辗转相除法(欧几里得算法)**：
求**最大公约数(`Greatest Common Divisor`)**
(1) **原理推导**

设我们要计算 `gcd(a, b)`，且 `a ≥ b > 0`。  
根据带余除法，存在唯一整数商 `q` 和余数 `r`，使得：
$$
a = b \cdot q + r, \quad 0 \le r < b
$$

**关键性质**：  
- 如果 `d` 能同时整除 `a` 和 `b`（即 `d | a` 且 `d | b`），则 `d` 必定也能整除 `r`（因为 `r = a - bq`）。  
- 反之，如果 `d` 能整除 `b` 和 `r`，则 `d` 也一定能整除 `a`。

因此，**`a` 和 `b` 的公约数集合与 `b` 和 `r` 的公约数集合完全相同**，所以它们的 **最大公约数也相等**：

$$
\gcd(a, b) = \gcd(b, r)
$$

其中 `r = a \b mod b`。

(2) **高效性**

- 每迭代一次，余数至少减半（更严格地，余数 `r < b/2` 或下一次迭代会快速减小），因此迭代次数为 `O(log min(a,b))`。
- 只需要整数除法和取模运算，非常适合编程实现。

(3) **特殊情况处理**

- 如果 `b = 0`，则 `gcd(a, 0) = a`（按定义，任何数整除 0）。
- 如果 `a = b = 0`，通常定义 GCD 为 0（视具体需求）。

#### 快速幂（二分求幂）
计算 $M^n$ 不需要做 n 次乘法，可以用快速幂将复杂度降到 $O(log n)$：

- 若指数为偶数：$ M^k = (M^{k/2})^2 $
- 若指数为奇数：$ M^k = M \cdot (M^{(k-1)/2})^2 $

每次指数减半，只需 $O(log n)$ 次矩阵乘法。

```cpp
vector<vector<long long>> matrixPower(const vector<vector<long long>> &M, int exp) {
        vector<vector<long long>> res = {{1, 0}, {0, 1}};
        vector<vector<long long>> base = M;
        while (exp > 0) {
            // 指数为奇数，提前面的 M
            if (exp & 1) {          // 等价于 exp % 2 == 0
                res = multiply(res, base);
            }
            // 指数为偶数次，进行 M = M^2
            base = multiply(base, base);
            // 位运算更新 exp
            exp >>= 1;              // 右移一位，等价于 exp %= 2;
        }
        return res;
    }
```

- **理解矩阵快速幂的代码**:
具体例子：**计算 $2^13$**（用矩阵类比）

1. **传统描述**：  
$2^{13} = 2 \cdot (2^2)^6$ （13 是奇数，先提一个 2）  
然后 $2^2$ 平方得 $2^4$，指数 6 是偶数，继续平方……  
最终把遇到的奇数次乘上去。

2. **二进制实现**：  
13 的二进制是 `1101`（从右到左：1、0、1、1）。  
- 最低位=1（奇数），`res *= base`（乘 2^1），`base` 平方成 2^2，指数右移为 `110` (6)。
- 下一位=0（偶数），不乘，`base` 平方成 2^4，指数右移为 `11` (3)。
- 下一位=1（奇数），`res *= base`（乘 2^4），`base` 平方成 2^8，指数右移为 `1` (1)。
- 下一位=1（奇数），`res *= base`（乘 2^8），`base` 平方成 2^16，指数右移为 `0`，退出。

`res` 先后乘了 $2^1$、$2^4$、$2^8$，乘积正好是 $2^{13}$。这完美对应 “若奇则先乘后平方，若偶则只平方” 的规则，只是换了个更直接的实现视角。

- **关于位运算(硬件层面一定更高效)**：
1. **`exp & 1`：等价于 `exp %2 == 1`，但是更高效**；计算过程：1 的二进制除了最低位是 1，其余所有高位全是 0。所以 `exp & 1` 实际上是**把 `exp` 的所有高位清零，只保留最低位的那一位，然后进行与运算**；
2. **`exp >>= 1`：右移一位，等价于 `exp %= 2`(实际编译器会该指令自动转化为右移运算)**；
