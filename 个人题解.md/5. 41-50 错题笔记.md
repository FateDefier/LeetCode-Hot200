## 41. 题号：209 -- 和为 K 的子数组（中等）
### 41.1 错误解法
```cpp
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        unordered_map<int, int> mp;     // preSum -- index

        int preSum = 0, len = 0;

        int n = nums.size();
        for (int i = 0; i < n; ++i) {
            preSum += nums[i];
            if (mp.find(target - preSum) != mp.end()) {
                len = i - mp[target - preSum] + 1;
            }
            mp[preSum] = i;
        }
        return len;
    }
};
```

- **思路**：找总和等于 `target` 的最短的子数组（**我题目看错了**），实际上就是找 `preSum[i] - preSum[j] = target` 且 `len = i - j + 1` 最小，不妨用哈希表存 `pair<preSum[j], j>`(`j` 为离 `i` 最近的值为 `target - preSum[i]` 的下标)，这时候 `preSum` 也不用数组;

- **错误点**：
1.  **算法原理误用**：
    -   试图用 `preSum[j] - preSum[i] == target` 来寻找子数组和。但这只能找到和**恰好等于** `target` 的子数组。
    -   题目要求是 `sum >= target`，且数组元素均为**正整数**。对于正整数数组，最优解法是利用单调性，而非哈希表查找精确值。
2.  **结果未取最小值**：
    -   `len = i - mp[...] + 1` 只是赋值，没有与历史最优解比较（缺少 `min` 操作）。
3.  **边界条件缺失**：
    -   未处理前缀和本身已经 `>= target` 的情况（即从下标0开始的子数组）。
4.  **时间复杂度退化**：
    -   即使逻辑修正为查找 `<= preSum - target`，`unordered_map` 也无法高效支持范围查询，最坏情况仍为 $O(n^2)$。

### 💡 拓展学习

| 场景 | 推荐方法 | 原因 |
| :--- | :--- | :--- |
| 正整数 + `>= target` | **滑动窗口** | 单调性保证窗口收缩的正确性，$O(n)$ |
| 含负数 + `== target` | **前缀和 + HashMap** | 失去单调性，需哈希记录历史前缀和 |
| 含负数 + `>= target` | **前缀和 + 单调队列/二分** | 需维护有序结构，$O(n \log n)$ |

### 41.2 正确解法及分析 -- 滑动窗口 正数（前缀和单调递增）最优
代码如下：
```cpp
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        int n = nums.size();
        int len = n + 1;    // 注意这里的空间优化，不用 INT_MAX
        int left = 0, sum = 0;

        for (int right = 0; right < n; ++right) {
            sum += nums[right];
            // 满足要求时，收缩左边界求最短子数组
            while (sum >= target) {
                // 更新长度
                len = min(len, right - left + 1);
                // 收缩左边界
                sum -= nums[left];
                ++left;
            }
        }
        return len == (n + 1) ? 0 : len;
    }
};
```

- **知识点**：**滑动窗口**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **思路**：滑动窗口常见思路，**左指针收缩边界保证最短，右指针探索**;

- **关键点**：
1. **`len = n + 1`** 而不是 `len = INT_MAX`：非常好的**空间优化**！；
2. **收缩左边界时，也要记得 `sum` 要减去左边界的值**；

### 41.3 正确解法 2 -- 前缀和 + 二分
```cpp
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        int n = nums.size();
        int len = n + 1;
        vector<int> preSum(n + 1, 0);        // 这里已经初始化了 preSum[0]，注意这里 preSum 长度为 n+1，因为 preSum[i] = nums[0..i-1]

        // 初始化 preSum
        for (int i = 0; i < n; ++i) {
            preSum[i + 1] = preSum[i] + nums[i];
        }

        // 二分搜索
        for (int i = 0; i < n; ++i) {
            int need = target + preSum[i];  // 注意这里是加号
            // 下面代码等价于：
            // auto it = lower_bound(preSum.begin() + i, preSum.end(), need);  // lower_bound >=，upper_bound <=
            // if (it != preSum.end()) {
            //     int curLen = it - preSum.begin() + 1;
            //     len = min(len, curLen);
            // }
            int left = i + 1, right = n;
            while (left <= right) {         // 等于不能省略，可能 left = right 且 preSum[left] = need
                int mid = (left + right) / 2;
                if (preSum[mid] >= need) {
                    len = min(len, mid - i);// 不用 + 1，因为 preSum[i] = nums[0..i-1] index 已经比 nums 大 1了
                    // 关键：继续向左收缩
                    right = mid - 1;
                } else {
                    left = mid + 1;
                } 
            }
        }
        return len == (n + 1) ? 0 : len;
    }
};
```

- **关键词**：**前缀和**，**二分查找**；

- **思路**：类似我的错误思路：找总和等于 `target` 的最短的子数组，实际上就是找 `preSum[i] - preSum[j] = target` 且 `len = i - j + 1` 最小，但要 **二分搜索** 找 `preSum[j]`;

### 41.4 正确解法 3 -- 前缀和 + 单调队列(可处理有负数情况，前缀和非单调)
```cpp
class Solution {
public:
    int minSubArrayLen(int target, vector<int>& nums) {
        int n = nums.size();
        vector<long long> preSum(n + 1, 0);     // 有负数时必须用 long long 防溢出
        for (int i = 0; i < n; ++i) {
            preSum[i + 1] = preSum[i] + nums[i];
        }

        deque<int> dq;          // 存单调递增(head-->tail)前缀和的 index，只维护候选左端点
        int len = n + 1;

        for (int j = 0; j <= n; ++j) {      // preSum 这里可以等
            // 1. 队首检查，当前前缀和 减(注意是减法) 最小前缀和
            while (!dq.empty() && preSum[j] - preSum[dq.front()] >= target) {
                len = min(len, j - dq.front()); // 不用 +1，preSum 下标比 nums 大 1
                dq.pop_front();
            }
            // 2. 更新左边界，即维护队尾
            // 如果队尾的元素即比 preSum[j] 大，index 又比 j 小，则出队，因为其候选左端点优先度永远小于 j 及其后续index
            while (!dq.empty() && preSum[dq.back()] >= preSum[j]) {
                dq.pop_back();
            }
            dq.push_back(j);
        }
        return len == (n + 1) ? 0 : len;
    }
};
```

- **关键词**：**前缀和**，**单调队列**；

- **思路**：由于有负数，**前缀和数组不单调**，故而用**单调递增队列**存对应前缀和的下标（不直接用队列存前缀和是因为排序也要用额外空间且思路不如这个清晰）；

## 42. 题号：53 -- 最大子数组和（中等）
### 42.1 错误解法
```cpp
class Solution {
public:
    int maxSubArray(vector<int> nums) {
        int n = nums.size();
        vector<int> preSum(n + 1, 0);
        int maxPreSum = -10000;
        
        int maxIdx = 0;
        for (int i = 0; i < n; ++i) {
            preSum[i + 1] = preSum[i] + nums[i];
            if (preSum[i + 1] > maxPreSum) {
                maxPreSum = preSum[i + 1];
                maxIdx = i + 1;
            }
        }

        int diff = maxPreSum - preSum[0];
        for (int i = 1; i <= n && i != maxIdx; ++i) {      // 遍历前缀和可以等于 n
            if (diff < maxPreSum - preSum[i]) {
                diff = maxPreSum - preSum[i];
            }
        }
        return diff;
    }
};
```

- **思路**：前缀和做差最大即为区间最大，先找到 `maxPreSum` ;

- **错误点**：
1. **严重逻辑错误**：忽略了要求 **最小前缀和的下标必须严格小于最大前缀和的下标**

### 42.2 正确解法 1 -- 前缀和
```cpp
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        int sum = 0;
        int max_sum = INT_MIN;
        int min_sum = 0;
        
        for (int num : nums) {
            sum += num;
            // 关键：用下标 <j 的 min_sum 更新 max_sum，从而保证max_sum 的下标永远在 min_sum 后面
            max_sum = max(max_sum, sum - min_sum);
            min_sum = max(min_sum, sum);
        }
        return max_sum;
    }
};
```

- **知识点**：**前缀和**；

- **关键点**：`max_sum = max(max_sum, sum - min_sum)` **用`index < j` 的 `min_sum` 更新 `max_sum`，从而保证max_sum 的下标永远在 min_sum 后面**，流式更新天然保证了时序！

### 42.3 正确解法 2 -- DP(空间优化) 最优
代码如下：
```cpp
class Solution {
public:
    int maxSubArray(vector<int>& nums) {
        int sum = nums[0];  // sum 维护截止当前最大子数组和
        int preSum = 0;     // preSum = dp[i - 1]

        for (int num : nums) {
            preSum = max(num, preSum + num);    // 要么加上当前，要么另起炉灶
            sum = max(sum, preSum);             // 更新 sum
        }
        return sum;
    }
};
```

- **知识点**：**动态规划**，**Kadane算法**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **思路**：状态转移 -- `dp[i] = max(nums[i], dp[i - 1] + nums[i])` (空间优化后，不用 dp[] 数组，直接用 sum 和 preSum)，**要么另起炉灶（`nums[i]` 单独成段），要么接在前面最大子数组后面**;

- **关键点**：使用 `sum` 和 `preSum` (空间$O(1)$)代替 `dp[]` 数组(空间 $O(n)$)，**但是这样就不能直接写状态转移方程了**；

### 42.4 正确解法 3 -- 分治递归（二维求最大子矩阵有用）
```cpp
class Solution {
private:
    int crossSum(vector<int>& nums, int L, int mid, int R) {
        // 扩展左边，求左边最大和
        int leftSum = INT_MIN; 
        int sum = 0;
        for (int left = mid; left >= L; --left) {
            sum += nums[left];
            leftSum = max(leftSum, sum);
        }
        // 扩展右边，求右边最大和
        int rightSum = INT_MIN;
        sum = 0;        // sum 重新置 0
        for (int right = mid + 1; right <= R; ++right) {
            sum += nums[right];
            rightSum = max(rightSum, sum);
        }
        return leftSum + rightSum;
    }
    int divide(vector<int>& nums, int L, int R) {
        // 递归基
        if (L == R) return nums[L];
        int mid = L + (R - L) / 2;      // 防溢出写法
        int leftMax = divide(nums, L, mid);
        int rightMax = divide(nums, mid + 1, R);
        int crossMax = crossSum(nums, L, mid, R);
        return max({leftMax, rightMax, crossMax});
    }
public:
    int maxSubArray(vector<int>& nums) {
        return divide(nums, 0, nums.size() - 1);
    }
};
```

- **知识点**：**分治法**，**递归**；

- **思路**：最大子数组要么全在左半部分，要么全在右半部分，要么跨过中点（要从中间向左右两边扩展）

- **时间复杂度 -- $O(n \cdot logn)$，空间复杂度 -- $O(n)$**

- **关键点**：
1. 穿过中点的最大和必须 **分别单独求左右两边的最大和**，以此保证穿过中点，直接一个循环 `L-->R` 不能保证最大和穿过中点；
2. **二分搜索用防溢出写法 `int mid = L + (R - L)/2`**；
3. **写递归时先弄清楚递归基**；

```cpp
class Solution {
public: 
    vector<int> productExceptSelf(vector<int>& nums) {
        int n = nums.size();
        vector<int> ans(n, 1);

        int preProduct = 1;
        for (int i = 0; i < n; ++i) {
            ans[i] *= preProduct;
            preProduct *= nums[i];
        }

        int postProduct = 1;
        for (int j = n - 1; j >= 0; --j) {
            ans[j] *= postProduct;
            postProduct *= nums[j];
        }

        return ans;
    }
};
```

- **知识点**：**前缀和/积**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **关键点**：**后缀乘积不用单独使用一个数组**，只用单个变量动态更新即可；

- **拓展**：如果允许除法，直接 `totalProduct/nums[i]`；

## 44. 题号：189 -- 轮转数组（中等）
### 44.1 解法 1 -- 额外空间 O(k)
```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int n = nums.size();
        k %= n;         // 关键：处理 k > n 的情况
        if (k == 0) return;
        
        vector<int> tmp(k, 0);

        for (int i = 0; i < k; ++i) {
            tmp[i] = nums[i + n - k];
        }

        // 把前面的元素向后移动 k
        for (int i = n - 1; i >= k; --i) {
            nums[i] = nums[i - k];
        }

        for (int i = 0; i < k; ++i) {
            nums[i] = tmp[i];
        }
    }
};
```

- **思路**：数组 `tmp[k]` 存末尾 `k` 个元素，然后，这 `k` 个元素前面所有元素后移，再把这 `k` 个元素赋值给前 `k` 个位置;

### 44.2 正确解法 1 -- 三次反转
代码如下：
```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int n = nums.size();
        k %= n;         // 关键：处理 k > n 的情况
        if (k == 0) return;
        
        // 第一次反转整体
        reverse(nums.begin(), nums.end());
        // 第二、三次反转前 k 个元素和其他元素
        reverse(nums.begin(), nums.begin() + k);
        reverse(nums.begin() + k, nums.end());
    }
};
```

- **知识点**：**三次反转**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$**

- **思路**：**第一次反转整体，第二反转前 k 个元素，第三次反转其他元素**;

### 44.3 正确解法 3 -- 环形替换法（做了解即可）
```cpp
class Solution {
public:
    void rotate(vector<int>& nums, int k) {
        int n = nums.size();
        k %= n;
        if (k == 0) return;

        int count = 0;      // 已处理的元素个数 -- 循环结束条件
        for (int start = 0; count < n; ++start) {
            int cur = start;
            int preVal = nums[start];
            do {
                int next = (cur + k) % n;       // 下一个目标位置
                int temp = nums[next];
                nums[next] = preVal;            // 右移 k 个位置
                // 更新 cur 和 preVal
                preVal = temp;
                cur = next;
                ++count;
            } while (cur != start);             // 回到环的起点终止
        }
    }
};
```

- **思路**：从位置 0 开始，每个元素跳到 `(i+k) % n`，沿着环走。当回到起点且遍历过所有元素时停止；

### 45.2 正确解法 1 -- 原地哈希
代码如下：
```cpp
class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        int n = nums.size();

        // 构建哈希表 nums[i] = i + 1
        for (int i = 0; i < n; ++i) {
            // 关键点：while 而非 if，因为交换来的正整数大概率与当前下标不满足 nums[i] = i + 1
            while (nums[i] >= 1 && nums[i] <= n && nums[i] != nums[nums[i] - 1]) {
                swap(nums[i], nums[nums[i] - 1]);
            }
        }

        // 再次遍历，找第一个不满足 nums[i] = i + 1 的数
        for (int i = 0; i < n; ++i) {
            if (nums[i] != i + 1) return i + 1;
        }
        // 最差情况 1~n 填满数组，则没在数组中出现过的最小正整数为 n+1
        return n + 1;
    }
};
```

- **知识点**：**原地哈希**；

- **时间复杂度 -- $O(n)$(均摊分析，每个元素定位只需要交换一次)，空间复杂度 -- $O(1)$**

- **思路**：原地哈希，`nums[i] = i + 1;` 对于一个长度为 `n` 的数组，答案范围是 `[1, n+1]`（最差情况 1~n 填满数组，则没在数组中出现过的最小正整数为 `n + 1`）;

- **关键点**：
1. 核心思路：**对于一个长度为 `n` 的数组，答案范围是 `[1, n+1]`**；
2. `for` 循环中，使用 `while` 而非 `if`，因为交换来的正整数大概率与当前下标不满足 `nums[i] = i + 1`；

### 45.3 解法 2 -- 标记法
```cpp
class Solution {
public:
    int firstMissingPositive(vector<int>& nums) {
        int n = nums.size();

        // 1.负数替换为 n + 1 做哨兵
        for (int i = 0; i < n; ++i) {
            if (nums[i] <= 0) nums[i] = n + 1;
        }

        // 2.对于 1 ~ n 出现过的数，把相应的 nums[i - 1] 标记为负数
        for (int i = 0; i < n; ++i) {
            int x = abs(nums[i]);       // 边标记边写(当处理下标 i 时，nums[i] 可能已经被前面的某个 j < i 标记为负数)，必须用绝对值
            if (x >= 1 && x <= n) {     // 排除哨兵，且只标记范围在 1~n 的正数
                if (nums[x - 1] > 0) nums[x - 1] = - nums[x - 1];                   // 这里可能改变哨兵
            }
        }

        // 3.找第一个值不为负数的下标(这里下标从最小开始找，碰到没有标记过的值对应的下标就是数组中没有出现过的最小正整数)
        for (int i = 0; i < n; ++i) {
            if (nums[i] > 0) return i + 1;      // 碰到哨兵 n + 1 说明未被标记，正常返回 i + 1
        }
        return n + 1;
    }
};
```

- **见注释**；

### 46.2 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    int longestConsecutive(vector<int>& nums) {
        if (nums.empty()) return 0;
        unordered_set<int> st(nums.begin(), nums.end());
        int res = 0;

        for (int x : st) {
            // 只有当 x 的前一个数 x - 1 不在集合中时，其才能作为起点
            if (st.find(x - 1) == st.end()) {
                int cur = x;
                int len = 1;
                // 看是否有 x(cur) + 1
                while (st.find(cur + 1) != st.end()) {
                    ++cur;
                    ++len;
                }
                res = max(res, len);
            }
        }
        return res;
    }
};
```

- **知识点**：**哈希集合**；

- **时间复杂度 -- $O(n)$（`while`循环处只有连续序列的起点才进入，$O(1)$），空间复杂度 -- $O(n)$**

- **关键点**：**只有当 `x` 的前一个数 `x - 1` 不在集合中时，其才能作为起点**；

## 47. 题号：49 -- 字母异位词分组（中等）
### 47.1 解法 1 -- 排序 + 哈希表
```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        if (strs.empty()) return {};
        unordered_map<string, vector<string>> mp;
        mp.reserve(strs.size());        // reserve 避免 rehash，养成习惯

        // 排序并根据 key 存入 value
        for (string &s : strs) {        // -- O(n)
            string key = s;
            sort(key.begin(), key.end());   // O(m logm)
            mp[key].push_back(s);       // 根据 key 存 s(value)
        }

        // 遍历 mp，把所有 value(vector<string>)存入 res
        vector<vector<string>> res;
        res.reserve(mp.size());
        for (auto& p : mp) {        // 使用引用，避免不必要的拷贝开销
            res.push_back(p.second);
        }
        return res;
    }
};
```

- **知识点**：**哈希表**，**排序**;

- **思路**：对每个字符串排序后相同的即为异位词，用 pair(key, value)存，其中 key 为排序后的字符串，value 为排序前的串 -- O(m logm)(排序)；

- **时间复杂度 -- $O(n \cdot m \cdot logm)$**(排序 $n \cdot logm$)

- **关键点**：**`reserve` 预分配避免重分配空间**；

### 47.2 解法 2 -- 计数编码作为 key
代码如下：
```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        if (strs.empty()) return {};
        unordered_map<string, vector<string>> mp;

        // 计数编码 "1#0#2#" 作为 key
        for (string &s : strs) {        // -- O(n)
            vector<int> cnt(26, 0);
            for (char &c : s) {         // -- O(m)
                ++cnt[c - 'a'];
            }
            // 编码
            string key;
            for (int i = 0; i < 26; ++i) {
                key += to_string(cnt[i]) + '#'; // to_string() + 字符串拼接 (+=) 涉及大量的内存分配、拷贝和格式化开销
            }
            mp[key].push_back(s);
        }

        vector<vector<string>> res;
        for (auto &p : mp) {
            res.push_back(p.second);
        }
        return res;
    }
};
```

- **知识点**：**哈希表**，**计数编码**；

- **时间复杂度 -- $O(n \cdot m)$**(遍历整个串 $O(m)$)

- **优化**：实际上，这套代码的时空表现均很差，需要优化一下几点：
1. **Key 的长度**：未优化版本的 Key 长度等于原字符串长度 $M$；未优化版本的 Key 长度约为 $26 \times (\text{平均数字位数}+1)$，对于短单词，未优化版本的 Key 反而**更长**。
2. **哈希计算**：`unordered_map` 需要对 Key 算哈希。Key 越长，哈希计算越慢，冲突概率也可能增加。未优化版本的长 Key 拖慢了哈希表的插入和查找速度。
3. **临时对象**：未优化版本在内层循环频繁创建 `vector<int>(26)` 和临时 `string`，对 GC/内存分配器压力更大。

**语法优化版本**：
```cpp
class Solution {
public:
    vector<vector<string>> groupAnagrams(vector<string>& strs) {
        if (strs.empty()) return {};
        unordered_map<string, vector<string>> mp;

        for (string &s : strs) {
            int cnt[26] = {0};
            for (char &c : s) ++cnt[c - 'a'];
            
            string key(26, 0);      // 关键优化：用定长字符数组作为 key，避免 to_string
            for (int i = 0; i < 26; ++i) {
                key[i] = static_cast<char>(cnt[i]);
            }
            mp[key].push_back(s);
        }

        vector<vector<string>> res;
        res.reserve(mp.size());

        for (auto &p : mp) {       
            res.push_back(p.second);
        }
        return res;
    }
};
```

## 48. 题号：242 -- 有效的字母异位词（简单）
### 48.1 正确解法及分析
代码如下：
```cpp
class Solution {
public:
    bool isAnagram(string s, string t) {
        if (s.length() != t.length()) return false;

        int cnt[26] = {0};
        // 遍历 s 对 cnt ++
        for (char &c : s) {
            ++cnt[c - 'a'];
        }
        // 遍历 t 对 cnt --
        for (char &c : t) {
            --cnt[c - 'a'];
            // 剪枝
            // if (cnt[c - 'a'] < 0) return false;
        }
        for (int i = 0; i < 26; ++i) {
            if (cnt[i] != 0) return false;
        }
        return true;
    }
};
```

- **知识点**：**频率计数**，**哈希表**；

- **时间复杂度 -- $O(n + m)$，空间复杂度 -- $O(1)$**(空间大小根据 `s` 和 `t` 包含的字符范围相应变化)

- **思路**：思路：分别遍历 s 和 t，遍历 s 时对 cnt ++，遍历 t 时 --，如果对应 cnt 数组所有值为 0，即为异位词;

- **提前剪枝可能更慢**：
1.  **分支预测惩罚**：`if (cnt[c-'a'] < 0)` 在正常异位词场景下**永远为 false**。但 CPU 无法100%确定，一旦误预测一次，流水线清空代价约 15-20 周期。而上述代码的第二个循环是纯顺序扫描，分支预测器准确率接近 100%。
2.  **数据依赖阻断流水线**：剪枝版本中 `--cnt[idx]` 和 `if (cnt[idx] < 0)` 存在**写后读 (RAW) 依赖**，CPU 必须等减法完成才能比较，无法乱序执行。上述代码的减法循环无任何依赖，现代 CPU 可每周期退休多条 store 指令。
3.  **剪枝收益极低**：对于真正的异位词（LeetCode 主流测试用例），剪枝**永远不会触发**。你为了一个极少出现的 early-exit 场景，付出了每次迭代额外的比较+跳转开销。这属于**悲观优化**。
4.  **编译器向量化机会**：上述代码的最终检查循环 `for(i=0;i<26;++i)` 极短且规整，编译器极易将其展开或用 SIMD 一条指令完成。剪枝版本的混合逻辑则难以向量化。

## 50. 题号：202 -- 快乐数（简单）
### 50.1 错误解法
```cpp
class Solution {
private:
    int isHappyInternal(int n) {
        // 递归基
        if (n == 1) return true;
        
        int m = 0;
        while (n / 10 > 0) {
            m += (n % 10) * (n % 10);
            n /= 10;
        }

        isHappyInternal(m);
        return false;
    }
public:
    bool isHappy(int n) {
        return isHappyInternal(n);
    }
};
```

- **思路**：**递归**;

- **错误点**：
1. **返回值丢失**：`isHappyInternal(m);` 没有 `return`，递归结果被丢弃；`isHappy(int n)` 也没有返回内部函数的结果。这会导致未定义行为2. **位数计算遗漏**：`while (n / 10 > 0)` 当 `n` 为个位数时直接退出循环，导致最高位（或仅剩的一位）的平方未被累加。应改为 `do-while` 或 `while (n > 0)`。
3. **无限递归风险**：快乐数问题中，非快乐数会进入**环**（如 4→16→37→...→4）。纯递归没有环检测机制，必然栈溢出。
4. **类型不匹配**：`isHappyInternal` 返回 `int`，但实际应返回 `bool`，语义混乱。


### 50.2 正确解法 1 -- 哈希表
代码如下：
```cpp
class Solution {
private:
    int next(int n) {
        int sum = 0;
        while (n > 0) {
            int digit = n % 10;
            sum += digit * digit;
            n /= 10;
        }
        return sum;
    }
public:
    bool isHappy(int n) {
        unordered_set<int> seen;        // 已经出现过的元素
        while (n != 1 && seen.find(n) == seen.end()) {
            seen.insert(n);
            n = next(n);
        }
        return n == 1;      // 不用单独判断 1，在这里统一判断
    }
};
```

- **知识点**：**哈希表**；

- **时间复杂度 -- $O(log \, n)$，空间复杂度 -- $O(n)$**

- **思路**：哈希表检测循环终止，碰到已经出现过的元素退出循环;

- **关键点**：
**循环一定会终止（不会无限增长）**，原因如下：

1. 对于 n 位数的数字，平方和最多为 `n×81`（每位最大 9²=81）(**这也是时间复杂度为 $O(log \, n)$(实际 $log_{10} n$) 的原因**)
2. 对于 `≥ 4` 位数，平方和会小于原数（`9999 → 324`）；对于 3 位数，平方和 `≤ 243`（**数值掉入三位数后，走 243 步必定能找到环或者到达 1，即这一部分时间复杂度为 $O(1)$**）
3. 所以最终一定进入 243 以内的小循环，要么到 1，要么陷入已知非快乐循环；

### 50.3 正确解法 2 -- 快慢指针(Floyd 判圈法)
```cpp
class Solution {
private:
    int next(int n) {
        int sum = 0;
        while (n > 0) {
            int digit = n % 10;
            sum += digit * digit;
            n /= 10;
        }
        return sum;
    }
public:
    bool isHappy(int n) {
        int slow = n;
        int fast = next(n);
        // fast = 1，循环结束，是快乐数，slow == fast 有环
        while (fast != 1 && slow != fast) {
            slow = next(slow);          // 慢指针每次走一步
            fast = next(next(fast));    // 快指针每次走两步
        }
        return fast == 1;
    }
};  
```

- **知识点**：**快慢指针**，**Floyd 判圈法**；

- **时间复杂度 -- $O(n)$，空间复杂度 -- $O(1)$(优势)**；

- **提醒**：快慢指针只能回答“**是否有环**”以及“**在哪里相遇**”，但是**不能记录完整的历史路径**；
